## 概述

这个视频用通俗易懂的方式（从 4 维向量开始举例），一步步讲清楚了"混合专家模型（MoE）"是什么、为什么需要它、以及它的核心工作原理。适合想理解 Qwen3、DeepSeek 等主流大模型名字里那些"235B A 22B"含义的初学者。

## 核心要点

- **MoE（Mixture of Experts）** 是混合专家模型，"Mo"=混合，"E"=专家——顾名思义，用多个"专家子网络"替代单一 FFN 层
- MoE 只替换 Transformer 中的 **FFN 层**，自注意力层不受影响
- 传统全连接层的参数量巨大（7B 模型可达几亿参数），想提升能力就得翻倍参数，计算量也跟着翻倍
- MoE 的核心思路：**参数量翻倍，但计算量不用翻那么多**——因为每个 token 只激活少数几个专家
- MoE 有两个关键步骤：① **怎么挑专家（路由/门控）** — 用一个 Router 网络给每个专家打分，选 Top-K；② **怎么计算** — 选中的专家加权求和
- 视频详细讲解了 Softmax 门控 vs Sigmoid 门控的区别，以及负载均衡的重要性
- 用"我爱吃苹果"这 4 个 token 的例子，手把手演示了整个 MoE 计算流程
- 最后解释了为什么 Qwen3-235B-A22B 中 235B 是总参数，22B 是激活参数——正因为用了 MoE

## 关键概念

| 概念 | 解释 |
|------|------|
| **全连接层 (FFN)** | Transformer 中参数量占比最高的部分，负责非线性变换 |
| **MoE** | 用多个"专家"FFN 替换单一 FFN，每个 token 只激活部分专家 |
| **专家 (Expert)** | 本质就是一个 FFN 子网络，不同专家擅长处理不同类型的 token |
| **门控网络 (Router/Gate)** | 决定每个 token 分配给哪些专家的小网络 |
| **Top-K 选择** | 从 N 个专家中选 K 个得分最高的来处理当前 token |
| **负载均衡** | 防止某些专家过载、某些闲置的技术，常见的有辅助损失和偏置调整 |
| **总参数 vs 激活参数** | 总参数 = 所有专家 + 共享参数；激活参数 = 每 token 实际计算的参数 |

## 一句话总结

> MoE 让模型"人多力量大"——参数量翻倍但计算量不大涨，关键是通过门控网络让每个 token 只找最合适的几个专家干活。
